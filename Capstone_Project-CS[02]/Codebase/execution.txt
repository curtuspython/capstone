# ==============================================================================
# CV Sorting using LLMs  -  CS[02]  -  Execution Instructions
# ==============================================================================
#
# WHAT THIS PROJECT DOES:
#   Reads a job description + a folder of candidate CVs, scores every CV
#   using two Google Gemini LLMs orchestrated by LangChain, computes
#   semantic similarity (Gemini embeddings -> TF-IDF cosine fallback),
#   extracts structured resume data via pyresparser (Tier 1) with spaCy
#   3.x NER as the automatic fallback (Tier 2), ranks candidates by a
#   weighted composite score, and exports results as TXT + CSV reports.
#   Supports an optional terminal-based interactive mode for real-time
#   query refinement.
#
# TECHNOLOGY STACK:
#   Google Gemini API   ->  LLM inference (gemini-2.5-flash + gemini-2.5-pro)
#   LangChain           ->  Prompt orchestration, score aggregation, structured
#                           output parsing (ChatPromptTemplate + JsonOutputParser)
#   LlamaIndex          ->  Semantic matching framework (embedding interface).
#                           Embedding calls go through google-genai SDK because
#                           llama-index-embeddings-gemini targets the deprecated
#                           v1beta endpoint that no longer serves text-embedding-004.
#   pyresparser (Tier 1)->  Structured information extraction from PDF/Word
#                           resumes (project requirement). Attempted first;
#                           NOTE: incompatible with spaCy 3.x (raises E053 due
#                           to outdated config.cfg format). Falls through to Tier 2.
#   spaCy 3.x (Tier 2)  ->  Automatic fallback for structured resume extraction
#                           via NER + regex. Compatible with Python 3.11+.
#   scikit-learn        ->  TF-IDF cosine similarity as semantic scoring Tier 2
#                           (used when Gemini text-embedding-004 returns 404).
#   --interactive mode  ->  Terminal-based query refinement (no GUI required)
#
# TWO LLMs USED (project requirement: minimum two LLMs):
#   LLM #1 : gemini-2.5-flash  ->  Analyses the job description ONCE
#                                   (fast; suitable for single-run structured extraction)
#   LLM #2 : gemini-2.5-pro    ->  Scores each candidate CV in a loop
#                                   (more capable; justified for accuracy-critical scoring)
#
# SUPPORTED FILE FORMATS:
#   Job description : .txt  |  .pdf  |  .docx
#   Candidate CVs   : .txt  |  .pdf  |  .docx  |  .md
#
# ==============================================================================
# VERSIONS
# ==============================================================================
#
#   Python           : 3.11 or higher (tested on 3.13)
#
#   LLM Models (Google Gemini):
#     LLM #1 : gemini-2.5-flash  (job description analysis  -- runs once)
#     LLM #2 : gemini-2.5-pro    (per-CV scoring loop       -- runs per candidate)
#
#   Core Python Packages:
#     google-genai             >=1.0.0   ->  Gemini API client (new SDK)
#     langchain                >=0.3.0   ->  Prompt orchestration & output parsing
#     langchain-google-genai   >=2.0.0   ->  LangChain <-> Gemini integration
#     llama-index-core         >=0.12.0  ->  LlamaIndex semantic framework
#     llama-index-llms-gemini  >=0.4.0   ->  LlamaIndex LLM wrapper for Gemini
#     llama-index-embeddings-gemini >=0.3.0  ->  LlamaIndex embedding wrapper
#     pyresparser              >=1.0.6   ->  Structured resume extraction (Tier 1)
#     nltk                     >=3.8.0   ->  Required by pyresparser
#     spacy                    >=3.0.0   ->  NER fallback extractor (Tier 2)
#       After install: python -m spacy download en_core_web_sm
#       (resume_parser.py auto-downloads on first run)
#     scikit-learn             >=1.3.0   ->  TF-IDF cosine semantic fallback
#     pypdf                    >=6.9.0   ->  PDF resume text extraction
#     python-docx              >=1.1.0   ->  Word (.docx) resume text extraction
#     fpdf2                    >=2.8.0   ->  PDF report generation

# STEP 4 - RUN THE PROJECT

#
#   BASIC USAGE:
#     python main.py --jd <path_to_job_description> --cvs <path_to_cv_directory>
#
#   FULL USAGE (all options):
#     python main.py \n#       --jd        <path_to_job_description_file>   \n#       --cvs       <path_to_cv_directory>           \n#       --output    <output_directory>               \n#       --min-score <0-100>                          \n#       --llm1      <gemini_model_for_jd_analysis>   \n#       --llm2      <gemini_model_for_cv_scoring>    \n#       --api-key   <GEMINI_API_KEY>                 \n#       --interactive
#
#   ARGUMENTS:
#     --jd          PATH   Job description file (.txt / .pdf / .docx)  [REQUIRED]
#     --cvs         PATH   Directory containing candidate CVs           [REQUIRED]
#     --api-key     KEY    Gemini API key (or use env var / .env file)  [OPTIONAL]
#     --output      DIR    Where to write TXT and CSV reports            [OPTIONAL, default: .]
#     --min-score   N      Flag candidates scoring below N (0-100)       [OPTIONAL]
#     --llm1        MODEL  Gemini model for JD analysis                  [OPTIONAL, default: gemini-2.5-flash]
#     --llm2        MODEL  Gemini model for CV scoring loop              [OPTIONAL, default: gemini-2.5-pro]
#     --interactive        Enter interactive mode after ranking           [OPTIONAL]
#
#   INTERACTIVE MODE (--interactive):
#   ----------------------------------
#   After ranking, enter a command loop for real-time refinement:
#     show <rank>         Inspect match explanation for a ranked candidate
#     filter <skill>      Keep only candidates whose CV mentions a skill
#     reset               Remove all filters and show full ranking
#     min-score <N>       Set minimum composite score threshold
#     edit-must <csv>     Edit must-have requirements (comma-separated)
#     edit-nice <csv>     Edit nice-to-have requirements (comma-separated)
#     edit-keywords <csv> Edit keywords (comma-separated)
#     rescore             Re-score candidates with updated requirements
#     rerank              Re-rank without re-scoring
#     export              Write TXT + CSV reports for current ranking
#     help / quit         Show help or exit interactive mode
#

# ==============================================================================
# SYSTEM ARCHITECTURE
# ==============================================================================
#
#   main.py              ->  CLI entry point; orchestrates the 4-step pipeline;
#                             includes --interactive mode for query refinement
#   llm_client.py        ->  Centralised Gemini client initialisation
#                             (google-genai SDK, LangChain, LlamaIndex wrappers)
#   resume_parser.py     ->  Two-tier resume extraction:
#                             Tier 1: pyresparser (project requirement)
#                             Tier 2: spaCy 3.x NER + regex (automatic fallback)
#                             + raw text extraction via pypdf / python-docx
#   jd_analyzer.py       ->  LLM #1 (gemini-2.5-flash) via LangChain chain
#                             -- structured JD extraction (must-have, nice-to-have,
#                             keywords, min experience, role summary)
#   cv_scorer.py         ->  LLM #2 (gemini-2.5-pro) via LangChain chain
#                             -- per-CV scoring on 4 dimensions + explanation
#   ranker.py            ->  Semantic scoring (three-tier fallback):
#                             Tier 1: Gemini text-embedding-004 cosine similarity
#                             Tier 2: TF-IDF cosine similarity (scikit-learn)
#                             Tier 3: neutral default 50.0
#                             + weighted composite score computation + ranking
#   report_generator.py  ->  Writes TXT + CSV reports to disk
#
