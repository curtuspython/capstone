# ==============================================================================
# CV Sorting using LLMs  -  CS[02]  -  Execution Instructions
# ==============================================================================
#
# WHAT THIS PROJECT DOES:
#   Reads a job description + a folder of candidate CVs, scores every CV
#   using two Google Gemini LLMs orchestrated by LangChain, computes
#   semantic similarity via LlamaIndex embeddings, extracts structured
#   resume data via pyresparser, ranks candidates by a weighted composite
#   score, and exports results as TXT + CSV reports. Supports an optional
#   terminal-based interactive mode for real-time query refinement.
#
# TECHNOLOGY STACK:
#   Google Gemini API   ->  LLM inference (gemini-2.5-flash / gemini-2.5-pro)
#   LangChain           ->  Prompt orchestration, score aggregation, structured
#                           output parsing (ChatPromptTemplate + JsonOutputParser)
#   LlamaIndex          ->  Semantic candidate-job matching via Gemini embeddings
#                           (text-embedding-004)
#   pyresparser         ->  Structured information extraction from PDF/Word
#                           resumes (name, skills, education, experience)
#   --interactive mode  ->  Terminal-based query refinement (no GUI required)
#
# TWO LLMs USED:
#   LLM #1 : gemini-2.5-flash  ->  Analyses the job description ONCE
#   LLM #2 : gemini-2.5-pro    ->  Scores each candidate CV in a loop
#
# SUPPORTED FILE FORMATS:
#   Job description : .txt  |  .pdf  |  .docx
#   Candidate CVs   : .txt  |  .pdf  |  .docx  |  .md
#
# ==============================================================================
# VERSIONS
# ==============================================================================
#
#   Python           : 3.11 or higher (tested on 3.13)
#
#   LLM Models (Google Gemini):
#     LLM #1 : gemini-2.5-flash  (job description analysis  — runs once)
#     LLM #2 : gemini-2.5-pro    (per-CV scoring loop       — runs per candidate)
#
#   Core Python Packages:
#     google-genai             >=1.0.0   ->  Gemini API client (new SDK)
#     langchain                >=0.3.0   ->  Prompt orchestration & output parsing
#     langchain-google-genai   >=2.0.0   ->  LangChain <-> Gemini integration
#     llama-index-core         >=0.12.0  ->  LlamaIndex framework
#     llama-index-embeddings-gemini >=0.3.0 -> Gemini embedding model
#     pyresparser              >=1.0.6   ->  Structured resume extraction
#     pypdf                    >=6.9.0   ->  PDF resume parsing
#     python-docx              >=1.1.0   ->  Word (.docx) resume parsing
#     fpdf2                    >=2.8.0   ->  PDF report generation
#

# STEP 4 - RUN THE PROJECT
# ==============================================================================
#
#   BASIC USAGE:
#     python main.py --jd <path_to_job_description> --cvs <path_to_cv_directory>
#
#   FULL USAGE (all options):
#     python main.py \
#       --jd    <path_to_job_description_file>  \
#       --cvs   <path_to_cv_directory>          \
#       --output  <output_directory>            \
#       --min-score <0-100>                     \
#       --llm1  <gemini_model_for_jd_analysis>  \
#       --llm2  <gemini_model_for_cv_scoring>   \
#       --api-key   <GEMINI_API_KEY>            \
#       --interactive
#
#   ARGUMENTS:
#     --jd          PATH   Job description file (.txt / .pdf / .docx)  [REQUIRED]
#     --cvs         PATH   Directory containing candidate CVs           [REQUIRED]
#     --api-key     KEY    Gemini API key (or use env var / .env file)  [OPTIONAL]
#     --output      DIR    Where to write TXT and CSV reports            [OPTIONAL, default: .]
#     --min-score   N      Flag candidates scoring below N (0-100)       [OPTIONAL]
#     --llm1        MODEL  Gemini model for JD analysis                  [OPTIONAL, default: gemini-2.5-flash]
#     --llm2        MODEL  Gemini model for CV scoring loop              [OPTIONAL, default: gemini-2.5-pro]
#     --interactive        Enter interactive mode after ranking to refine [OPTIONAL]
#                          criteria, filter candidates, and re-run sorting
#
#   INTERACTIVE MODE (--interactive):
#   ----------------------------------
#   After the initial ranking, the terminal enters a command loop where
#   you can refine results without restarting the pipeline:
#     show <rank>        Inspect match explanation & evidence for a candidate
#     filter <skill>     Filter candidates whose CV contains a given skill
#     reset              Remove all filters and show full ranking
#     min-score <N>      Set minimum composite score threshold
#     edit-must <csv>    Edit must-have requirements (comma-separated)
#     edit-nice <csv>    Edit nice-to-have requirements (comma-separated)
#     edit-keywords <csv> Edit keywords (comma-separated)
#     rescore            Re-score candidates with updated requirements
#     rerank             Re-rank (no re-scoring) with current settings
#     export             Write TXT + CSV reports for the current ranking
#     help / quit        Show help or exit interactive mode
#

# ==============================================================================
# SYSTEM ARCHITECTURE
# ==============================================================================
#
#   main.py              ->  CLI entry point; orchestrates the full pipeline;
#                             includes --interactive mode for query refinement
#   llm_client.py        ->  Centralised Gemini client (google-genai,
#                             LangChain ChatGoogleGenerativeAI, LlamaIndex
#                             GeminiEmbedding)
#   resume_parser.py     ->  Extracts raw text (pypdf / python-docx) +
#                             structured data (pyresparser) from resumes
#   jd_analyzer.py       ->  LLM #1 via LangChain — structured JD extraction
#   cv_scorer.py         ->  LLM #2 via LangChain — per-CV scoring with
#                             structured resume data + parsed JD as input
#   ranker.py            ->  Weighted composite scoring + LlamaIndex semantic
#                             similarity via Gemini embeddings
#   report_generator.py  ->  Writes TXT + CSV reports to disk
#
